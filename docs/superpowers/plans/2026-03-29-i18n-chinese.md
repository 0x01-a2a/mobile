> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Wire the existing i18next infrastructure into all redesigned screens and add complete zh-CN translations so the app fully supports Chinese.

**Architecture:** The i18n system (`src/i18n.ts`, `src/locales/en.json`, `src/locales/zh-CN.json`) and its App.tsx initialization are already in place. This plan: (1) adds all new locale keys with Chinese translations, (2) adds a Jest mock so existing tests keep passing, (3) wires `useTranslation` + `t()` into every screen, (4) adds a language switcher in You.tsx Settings tab.

**Tech Stack:** React Native 0.84.1, TypeScript, i18next, react-i18next, Jest.

The existing tests check for exact English strings like `'No new jobs'`, `'Pay & Accept'`, etc. After this plan wires `t()` into the screens those strings become `t('inbox.noNewJobs')`. Without a mock, Jest will receive the raw key string and tests will fail. The mock must load `en.json` and do a real key lookup so `t('inbox.noNewJobs')` returns `'No new jobs'`.

- [ ] **Step 1: Create `__mocks__/react-i18next.js`**

```javascript
// __mocks__/react-i18next.js
const en = require('../src/locales/en.json');

function lookup(obj, path) {
  return path.split('.').reduce((cur, k) => (cur && typeof cur === 'object' ? cur[k] : undefined), obj);
}

function t(key, params) {
  let val = lookup(en, key);
  if (typeof val !== 'string') return key;
  if (params) {
    Object.entries(params).forEach(([k, v]) => {
      val = val.replace(new RegExp(`\\{\\{${k}\\}\\}`, 'g'), String(v));
    });
  }
  return val;
}

const useTranslation = () => ({
  t,
  i18n: { language: 'en', changeLanguage: jest.fn().mockResolvedValue(undefined) },
});

module.exports = {
  useTranslation,
  Trans: ({ children }) => children,
  initReactI18next: { type: '3rdParty', init: jest.fn() },
  t,
};
```
